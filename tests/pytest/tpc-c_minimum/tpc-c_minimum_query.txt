必要なテーブル・初期データはすでに投入済みとする。

// TPC-C New-Order の最小クエリ構成
1. `START TRANSACTION`
2. `SELECT` 顧客 (`bmsql_customer`)          -- 存在確認
3. `SELECT` 倉庫 (`bmsql_warehouse`)         -- 税率取得
4. `SELECT ... FOR UPDATE` 地区 (`bmsql_district`) -- `d_next_o_id` をロック取得付きで取得
5. `UPDATE` 地区 `d_next_o_id = d_next_o_id + 1`
6. `INSERT` 受注ヘッダ (`bmsql_oorder`)
7. `INSERT` 新規受注キュー (`bmsql_new_order`)
8. 各注文行ごとに:
   - `SELECT` 商品 (`bmsql_item`)
   - `SELECT ... FOR UPDATE` 在庫 (`bmsql_stock`)
   - `UPDATE` 在庫数量・累積値 (`bmsql_stock`)
   - `INSERT` 注文明細 (`bmsql_order_line`)
9. 正常系: `COMMIT`
   例外系 (商品未検出など): `ROLLBACK`

// TPC-C Delivery の最小クエリ構成
1. `START TRANSACTION`
2. `SELECT ... FOR UPDATE` 新規注文 (`bmsql_new_order`) -- 指定倉庫・地区の最古 `no_o_id`
3. `DELETE` `bmsql_new_order` 該当行
4. `SELECT` `bmsql_oorder` -- `o_c_id` 取得
5. `UPDATE` `bmsql_oorder` -- `o_carrier_id` 更新
6. `UPDATE` `bmsql_order_line` -- `ol_delivery_d = NOW()`
7. `SELECT SUM(ol_amount)` 注文明細合計
8. `UPDATE` `bmsql_customer` -- `c_balance` 加算・`c_delivery_cnt` インクリメント
9. `COMMIT`（失敗時 `ROLLBACK`）

// TPC-C Order-Status の最小クエリ構成
1. `START TRANSACTION`
2. `SELECT` 顧客候補一覧 (`bmsql_customer`) -- `c_last` 一致・`ORDER BY c_first`
3. 選択顧客に対し `SELECT` 最新注文 (`bmsql_oorder` `ORDER BY o_id DESC LIMIT 1`)
4. `SELECT` 注文明細 (`bmsql_order_line`)
5. `COMMIT`（失敗時 `ROLLBACK`）

// TPC-C Payment の最小クエリ構成
1. `START TRANSACTION`
2. `SELECT` 倉庫 YTD (`bmsql_warehouse`)・地区 YTD (`bmsql_district`)・顧客情報 (`bmsql_customer`)
3. `UPDATE` `bmsql_warehouse` -- `w_ytd += amount`
4. `UPDATE` `bmsql_district`  -- `d_ytd += amount`
5. `SELECT ... FOR UPDATE` `bmsql_customer` -- `c_balance` 取得
6. `UPDATE` `bmsql_customer` -- 残高減算・`c_ytd_payment`/`c_payment_cnt` 更新
7. `INSERT` `bmsql_history`
8. `COMMIT`（失敗時 `ROLLBACK`）

// TPC-C Stock-Level の最小クエリ構成
1. `START TRANSACTION`
2. `SELECT` `bmsql_district` -- `d_next_o_id` 取得
3. `SELECT COUNT(DISTINCT s_i_id)` -- `bmsql_order_line` と `bmsql_stock` を結合し低在庫数を集計
4. `COMMIT`（失敗時 `ROLLBACK`）
